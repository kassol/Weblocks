# PROTOTYPE — 首次体验验收记分卡

问题：哪一小组可观察标准，足以判断首要玩家能独立开始、5 分钟内获得成果，并通过 30 分钟内完成首个挑战证明理解核心拼搭交互？

本原型把一次人工观察记录交给纯函数判定，再用故意天真的“只看总体平均值”模型对照，暴露成人帮助、输入方式差异和重复阻塞被平均数掩盖的情况。

候选验收轮次：至少 6 名首次玩家，鼠标与触控各至少 3 名；所有有效观察都必须同时通过全部标准，没有同一阻塞重复出现两次。

观察员只做设备设置并计时，不解释操作、不碰设备。产品内提示允许使用且只作诊断记录；技术故障作废并补测。记录只使用测试编号、年龄段、输入方式、三个时间点、成人干预次数和阻塞原因，不需要账号、埋点、录像或个人身份。

运行：

```sh
node prototype-first-session-acceptance/cli.mjs
```

最小检查：

```sh
node prototype-first-session-acceptance/cli.mjs --check
```

这是一次性决策原型，不是生产分析、用户研究平台或自动化验收框架。
